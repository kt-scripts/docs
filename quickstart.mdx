This quickstart takes you from a fresh installation to a fully working KT Extra Menu in just a few minutes. You select your framework, configure the menu system, set the command and keybind, and test the script directly in game – no deep prior knowledge required.

<Tip>
  Make sure KT Extra Menu is already installed before starting this quickstart. If not, follow the [installation guide](/installation) first.
</Tip>

## Quick configuration

<Tabs>
  <Tab title="ESX">
    Minimal configuration for ESX servers:

    ```lua
    Config.Framework  = 'esx'
    Config.MenuSystem = 'ox'
    ```
  </Tab>
  <Tab title="QBCore / QBox">
    Minimal configuration for QBCore and QBox servers:

    ```lua
    Config.Framework  = 'qbcore'   -- or 'qbox' for QBox
    Config.MenuSystem = 'ox'
    ```
  </Tab>
</Tabs>

## Setup steps

<Steps>
  <Step title="Select framework">
    Open `resources/kt_extra_menu/config.lua` and set `Config.Framework` to the framework used by your server. Possible values:

    ```lua
    -- Choose one of the following frameworks:
    Config.Framework = 'esx'        -- ESX
    Config.Framework = 'qbcore'     -- QBCore
    Config.Framework = 'qbox'       -- QBox
    Config.Framework = 'standalone' -- No framework (vanilla FiveM)
    ```

    With `standalone`, framework-dependent features such as job restrictions and framework-based player notifications are not available.
  </Step>
  <Step title="Select menu system">
    Decide which menu interface players will see. KT Extra Menu supports two systems:

    ```lua
    Config.MenuSystem = 'ox'      -- ox_lib context menu (default, recommended)
    Config.MenuSystem = 'lation'  -- Lation Scripts Modern UI
    ```

    The **Lation Modern UI** offers a more visually appealing interface but requires a separate installation of the Lation UI resource (available via the [Lation Scripts Discord](https://discord.gg/A6UgrEhc8G)).
  </Step>
  <Step title="Set command and keybind">
    Define the chat command and key combination players can use to open the menu:

    ```lua
    Config.Command = 'vehiclemenu' -- Chat command: /vehiclemenu
    Config.Keybind = 'F11'         -- Default keybind (player-customizable)
    ```

    Players can customize the keybind at any time in the GTA V settings under **Key bindings → FiveM**.
  </Step>
  <Step title="Restart server">
    Save `config.lua` and restart the server, or run the following in the server console:

    ```cfg
    restart kt_extra_menu
    ```

    Check the console output for errors. A successful initialization is confirmed by a status message from KT Extra Menu.
  </Step>
  <Step title="Test in game">
    Connect to the server, get into a vehicle and open the menu in one of the following ways:

    - **Chat command:** type `/vehiclemenu` and confirm
    - **Keybind:** press `F11` (default)

    The menu shows all available liveries as well as the configured extras for the currently used vehicle. Select a livery or toggle an extra – changes are immediately visible on the vehicle.

    <Tip>
      If `Config.OnlyDriver = true` is set, **only drivers** can open the menu. Passengers will receive an error message in this case. Make sure you're in the driver's seat when testing the script.
    </Tip>
  </Step>
</Steps>

## Common quickstart issues

| Issue | Possible cause | Solution |
| --- | --- | --- |
| Menu does not open | `ox_lib` not started or wrong startup order | Check order in `server.cfg`: `ensure ox_lib` before `ensure kt_extra_menu` |
| Unknown command | Script did not start correctly | Check console output for errors; run `restart kt_extra_menu` |
| Liveries are not displayed | Vehicle has no liveries defined | Add livery entries in `config.lua` for the relevant vehicle model |
| Job check fails | Wrong framework selected | Set `Config.Framework` in `config.lua` to the framework actually in use |

## Further steps

Once the script is working in principle, you can continue with the detailed configuration:

- **Configure liveries** – assign liveries with custom names to vehicle models
- **Job restrictions** – limit access to specific jobs
- **Change language** – switch between German and English via `Config.Language`
- **Use exports** – integrate KT Extra Menu into your own resources via the Lua exports (`OpenVehicleMenu`, `OpenLiveryMenu`, `OpenExtrasMenu`)

For questions, the [Lation Scripts Discord](https://discord.gg/A6UgrEhc8G) is at your disposal.
