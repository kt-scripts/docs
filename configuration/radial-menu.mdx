With the Radial Menu integration, the entry for the vehicle menu appears automatically in the radial menu as soon as you sit in a vehicle as the driver – without having to press a keybind or type a command. This is especially useful when your server already uses a radial menu and you want to bundle all vehicle actions in one central place. KT Extra Menu supports both **ox\_lib Radial** and **qb-radialmenu**.

<Note>
  As soon as `Config.UseRadial = true` is set, the keybind configured in `Config.Keybind` is automatically disabled. The menu can then only be opened via the radial menu or the chat command.
</Note>

## Setup

<Tabs>
  <Tab title="ox_lib Radial">
    Use this configuration if your server uses **ox\_lib** as the radial menu:

    ```lua
    Config.UseRadial = true
    Config.OxRadial = true
    Config.RadialIcon = 'car'
    ```

    The radial menu element appears automatically with the specified FontAwesome icon as soon as the player is in a vehicle as the driver. Make sure `ox_lib` is started before `kt_extra_menu` in your `server.cfg`.

    ```cfg
    ensure ox_lib
    ensure kt_extra_menu
    ```
  </Tab>
  <Tab title="qb-radialmenu">
    Use this configuration if your server uses **qb-radialmenu**:

    ```lua
    Config.UseRadial = true
    Config.OxRadial = false
    Config.QBRadialId = 'vehiclecustomizer'
    ```

    The `QBRadialId` value is used as the unique ID for the entry in qb-radialmenu. Adjust it if needed in case of conflicts with other entries.
  </Tab>
</Tabs>

## Configuration options

<ParamField path="Config.UseRadial" default="false" type="boolean">
  Enables the radial menu integration. When `true`, the keybind is disabled and the menu appears as an entry in the radial menu instead.
</ParamField>

<ParamField path="Config.OxRadial" default="false" type="boolean">
  Determines which radial menu system is used. Only relevant when `Config.UseRadial = true`.

  | Value | System |
  | --- | --- |
  | `true` | ox\_lib Radial Menu |
  | `false` | qb-radialmenu |
</ParamField>

<ParamField path="Config.RadialIcon" default="'car'" type="string">
  The name of the **FontAwesome** icon displayed next to the menu entry in the radial menu. Only relevant when `Config.OxRadial = true`. Examples: `'car'`, `'wrench'`, `'paint-brush'`.
</ParamField>

<ParamField path="Config.QBRadialId" default="'vehiclecustomizer'" type="string">
  The unique ID of the entry in qb-radialmenu. Only relevant when `Config.OxRadial = false`. Make sure this ID is not duplicated on your server.
</ParamField>

## When does the radial menu entry appear?

The entry is only shown under the following conditions:

- The player sits **as the driver** in the vehicle (if `Config.OnlyDriver = true`).
- The player has the **required job** (if `Config.RequiredJob` is not empty).
- The script is correctly started and `Config.UseRadial = true`.

If the player exits the vehicle or switches to the passenger seat, the entry is automatically removed from the radial menu.

---

## Command only – no radial, no keybind

If you don't want to use either a radial menu or a keybind and the menu should only be accessible via the chat command, use the following values:

```lua
Config.UseRadial = false
Config.Keybind = false
```

The menu is then only accessible via `/vehiclemenu` (or the command configured in `Config.Command`).
