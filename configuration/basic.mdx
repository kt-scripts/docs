The central configuration file of KT Extra Menu is located in the resource folder at `config.lua`. All fundamental behaviors of the script – framework selection, language, menu system, access restrictions and notifications – are controlled here. Adjust this file to your server setup before starting the script.

## Complete basic configuration block

```lua
Config = {}

-- Framework selection: 'esx', 'qbcore', 'qbox', 'standalone'
Config.Framework = 'qbox'

-- Language selection: 'de', 'en'
Config.Language = 'en'

-- 'ox' ox_lib context menu
-- 'lation' Lation Scripts Modern UI
Config.MenuSystem = 'ox'

-- Automatically close menu after selection
Config.CloseMenuAfterSelection = false

-- Command to open the menu
Config.Command = 'vehiclemenu'

-- Keybind to open menu (only works if Config.UseRadial is false)
-- Set to false to disable keybind completely
Config.Keybind = 'F11'

-- Only the driver can use the menu
Config.OnlyDriver = true

-- Examples: 'admin', 'mechanic', 'police' nothing and everyone can use it
Config.RequiredJob = {}

-- Notification settings
Config.Notifications = {
    duration = 3000,
    position = 'top-right'
}
```

## Configuration options

### General

<ParamField path="Config.Framework" type="string" required>
  Specifies the framework in use. Possible values:

  | Value | Framework |
  | --- | --- |
  | `'esx'` | ESX |
  | `'qbcore'` | QBCore |
  | `'qbox'` | QBox |
  | `'standalone'` | No framework – no job check |

  **Default:** `'qbox'`
</ParamField>

<ParamField path="Config.Language" default="'en'" type="string">
  Language for menu texts and notifications. Available locale files are located under `locales/`.

  | Value | Language |
  | --- | --- |
  | `'en'` | English |
  | `'de'` | German |
</ParamField>

<ParamField path="Config.MenuSystem" default="'ox'" type="string">
  Selects the menu system for the user interface.

  | Value | System |
  | --- | --- |
  | `'ox'` | ox\_lib context menu |
  | `'lation'` | Lation Scripts Modern UI |

  More details about the UI systems can be found under [UI settings](/configuration/ui-settings).
</ParamField>

<ParamField path="Config.CloseMenuAfterSelection" default="false" type="boolean">
  When set to `true`, the menu is automatically closed after each selection. When `false`, the menu stays open so multiple changes can be made in sequence.
</ParamField>

---

### Opening the menu

<ParamField path="Config.Command" default="'vehiclemenu'" type="string">
  Chat command used to open the vehicle menu. The command is always active, regardless of keybind or radial menu. Example: `/vehiclemenu`
</ParamField>

<ParamField path="Config.Keybind" default="'F11'" type="string | false">
  Key used to open the menu. Only works if `Config.UseRadial = false`. Set this value to `false` to disable the keybind completely.

  ```lua
  Config.Keybind = 'F11'   -- Keybind active
  Config.Keybind = false   -- Keybind disabled
  ```
</ParamField>

---

### Access permissions

<ParamField path="Config.OnlyDriver" default="true" type="boolean">
  When `true`, only the driver of the vehicle can open the menu. Passengers and other occupants have no access.
</ParamField>

<ParamField path="Config.RequiredJob" default="{}" type="table">
  List of jobs allowed to use the menu. An empty table (`{}`) means that all players have access (provided `Config.OnlyDriver` and other conditions are met).

  ```lua
  -- Everyone may use the menu
  Config.RequiredJob = {}
  
  -- Only one specific job
  Config.RequiredJob = {'mechanic'}
  
  -- Multiple jobs
  Config.RequiredJob = {'mechanic', 'police', 'ambulance'}
  ```

  <Note>
    With `Config.Framework = 'standalone'`, the job check is skipped – all players have unrestricted access.
  </Note>
</ParamField>

---

### Notifications

<ParamField path="Config.Notifications.duration" default="3000" type="number">
  Display duration of notifications in milliseconds. `3000` equals 3 seconds.
</ParamField>

<ParamField path="Config.Notifications.position" default="'top-right'" type="string">
  Screen position of notifications (only relevant with `Config.MenuSystem = 'ox'`).

  | Value | Position |
  | --- | --- |
  | `'top-right'` | Top right |
  | `'top-left'` | Top left |
  | `'bottom-right'` | Bottom right |
  | `'bottom-left'` | Bottom left |
</ParamField>
