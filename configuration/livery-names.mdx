By default, KT Extra Menu shows the internal name of a paint job stored by the game. With `Config.CustomLiveryNames` you can assign custom, meaningful names for each vehicle model – for example to clearly label service vehicles or to name liveries in a way that makes sense in roleplay. The custom names appear in the livery selection of the menu instead of the in-game labels.

<Note>
  Model names must be specified in **lowercase** (e.g. `'police'`, not `'POLICE'`). The livery index starts at **0** for the first paint job.
</Note>

## Configuration example

```lua
Config.CustomLiveryNames = {
    ['police'] = {
        [0] = 'LSPD Black/White',
        [1] = 'LSPD Slicktop',
        [2] = 'Sheriff',
    },
    ['ambulance'] = {
        [0] = 'Los Santos Medical',
        [1] = 'Fire Department',
    },
}
```

Add a separate block for each vehicle model. The keys are model names, the values are tables of index numbers and the associated labels.

## Fallback logic

KT Extra Menu uses the following priority when displaying livery names:

1. **Custom name** from `Config.CustomLiveryNames` (highest priority)
2. **In-game livery name**, if the vehicle model has one
3. **Fallback label** `Livery N` (where `N` is the index plus 1) if neither a custom nor an in-game name is available

So you don't have to assign a name to every paint job – only where you need a custom label.

<Tip>
  The easiest way to find a vehicle's model name is to get into the vehicle in game and run `print(GetEntityModel(GetVehiclePedIsIn(PlayerPedId(), false)))` in the F8 console – or look it up directly in your framework's vehicle database. Model names must always be entered in **lowercase** in the config, e.g. `'police'`, `'ambulance'`, `'firetruck'`.
</Tip>

## Vehicle extras

In addition to liveries, KT Extra Menu also supports **vehicle extras** (e.g. spoilers, light bars, additional parts). These are detected **fully automatically** – the script checks extras 0 to 20 and shows all available extras in the menu.

**No manual configuration** for extras is required. Available extras are loaded dynamically for the currently used vehicle and can be toggled on and off directly in the menu under the **Extras** tab.
