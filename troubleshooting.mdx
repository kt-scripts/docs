On this page you'll find solutions for the most common issues with **KT Extra Menu**. Go through the relevant sections before asking for help on the support Discord – most problems can be resolved quickly yourself.

<AccordionGroup>
  <Accordion title="The menu does not open">
    If the menu doesn't react after typing the command or pressing the keybind, check the following:

    - **Are you in a vehicle?** The menu only works inside a vehicle.
    - **Are you the driver?** If `Config.OnlyDriver = true` is set, only the driver of the vehicle can open the menu.
    - **Do you have the required job?** If `Config.RequiredJob` is populated, your active job must match one of the listed jobs.
    - **Is `ox_lib` installed and started?** `ox_lib` is a mandatory dependency and must be started **before** `kt_extra_menu` in `server.cfg`:

    ```cfg
    ensure ox_lib
    ensure kt_extra_menu
    ```

    - Check the **server console** for error messages emitted directly during the startup of `kt_extra_menu`.
  </Accordion>

  <Accordion title="Command /vehiclemenu not found">
    If the chat command `/vehiclemenu` (or the command set in `Config.Command`) does not exist:

    - Make sure the resource is correctly listed in your `server.cfg`:

    ```cfg
    ensure ox_lib
    ensure kt_extra_menu
    ```

    - The **folder name** of the resource must exactly match the resource name. If the folder is, for example, called `KT-Extra-Menu`, the entry would be:

    ```cfg
    ensure KT-Extra-Menu
    ```

    - Restart the server and check whether the resource is reported as started in the console.
  </Accordion>

  <Accordion title="Liveries are not displayed">
    The menu shows the message **"This vehicle has no liveries!"** – this is not an error, but the expected behavior for vehicles without liveries.

    - **Not every vehicle has liveries.** Test the menu with a vehicle known to have liveries (e.g. `police`).
    - **Custom livery names:** When assigning custom names in `Config.CustomLiveryNames`, the vehicle model name must be **lowercase**:

    ```lua
    Config.CustomLiveryNames = {
        ['police'] = {       -- ✅ Correct: lowercase
            [0] = 'LSPD Black/White',
            [1] = 'LSPD Slicktop',
            [2] = 'Sheriff',
        },
        -- ['POLICE'] = { ... }  ❌ Wrong: uppercase
    }
    ```

    - The livery index starts at `0`.
  </Accordion>

  <Accordion title="Extras are not displayed">
    The menu shows the message **"This vehicle has no extras!"** – this is also expected behavior.

    - **Not every vehicle has extras.** KT Extra Menu automatically scans extra slots 0 to 20. If none exist, the message is shown.
    - Some vehicles only have extras in certain mods or DLC variants.
    - Test the menu with a vehicle known to have extras to verify that the feature works in principle.
  </Accordion>

  <Accordion title="Job restriction does not take effect">
    If all players can open the menu even though `Config.RequiredJob` is set:

    - Check the **exact spelling** of the job name – it must match 1:1 with the job name in your framework (case sensitive).
    - When `Config.Framework = 'standalone'` is set, job restrictions are **generally ignored**, since there is no framework context.
    - Make sure `Config.Framework` matches the framework actually installed:

    ```lua
    -- Possible values: 'esx', 'qbcore', 'qbox', 'standalone'
    Config.Framework = 'esx'
    ```
  </Accordion>

  <Accordion title="Keybind does not work">
    If the configured keybind (default: `F11`) doesn't respond:

    - The keybind is **only active when `Config.UseRadial = false`** is set. When the radial menu is enabled, the keybind is automatically disabled.
    - Check that `Config.Keybind` is not set to `false`:

    ```lua
    Config.Keybind = 'F11'   -- ✅ Keybind active
    -- Config.Keybind = false  -- ❌ Keybind disabled
    ```

    - Try rebinding the key in the **GTA V settings → Key bindings**, in case it conflicts with another script.
  </Accordion>

  <Accordion title="Radial menu entry does not appear">
    If the entry is missing from the radial menu:

    - `Config.UseRadial` must be set to `true`:

    ```lua
    Config.UseRadial = true
    ```

    - You must be **sitting in a vehicle** for the entry to appear.
    - If `Config.OnlyDriver = true`, you must also be the **driver** of the vehicle.
    - Check whether `Config.OxRadial` correctly points to your installed radial menu resource:

    ```lua
    Config.OxRadial = true   -- ox_lib Radial Menu
    -- Config.OxRadial = false  -- qb-radialmenu
    ```

    - When using `qb-radialmenu`: Make sure `Config.QBRadialId` is unique and not used by another resource.
  </Accordion>

  <Accordion title="Lation Modern UI does not work">
    If the menu stays empty or does not appear at all when using Lation Modern UI:

    - The resource **`modernui`** (Lation Scripts Modern UI) must be installed and started in `server.cfg` before `kt_extra_menu`.
    - Make sure `Config.MenuSystem` is set correctly:

    ```lua
    Config.MenuSystem = 'lation'
    ```

    - Check the **server console and the browser console (F8)** for error messages related to `modernui` or `kt_extra_menu`.
    - The `Config.LationUI` settings are only used when `Config.MenuSystem = 'lation'` is active:

    ```lua
    Config.LationUI = {
        header    = 'Vehicle Customization',
        subheader = 'Customize your vehicle',
    }
    ```
  </Accordion>

  <Accordion title="Version check error in the console">
    If a message like `Version check currently disabled due to a GitHub issue` appears on server start:

    - This is a **non-critical warning**. The script was unable to reach the external version server (GitHub).
    - The **functionality of the script is not affected** – all features continue to work normally.
    - The warning may appear when GitHub is temporarily unreachable or the server has no outgoing HTTP access.
    - The version check automatically retries every 60 minutes.

    <Note>
      If you want to disable the version check entirely, please contact the support Discord. Note that without an active check you will no longer receive automatic update notices.
    </Note>
  </Accordion>
</AccordionGroup>

<Note>

  - The relevant section of your **`server.cfg`** (the `ensure` lines)
  - Your full **`config.lua`**
  - All **error messages** from the server console and/or the client console (F8)
